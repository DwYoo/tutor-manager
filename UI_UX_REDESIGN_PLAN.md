## 1. 현재 UI/UX 진단 요약

### 1.1 전체 평가

| 영역 | 현재 점수 (10점) | 핵심 이슈 |
|------|:---:|------|
| 비주얼 디자인 | 6.5 | 깔끔하나 밋밋, 시각적 계층 부족, 차트 표현력 제한 |
| 정보 구조 (IA) | 5.5 | 학생 상세 4탭x10서브탭 과부하, 깊은 뎁스 |
| 인터랙션 디자인 | 6.0 | 드래그/컨텍스트 메뉴 있으나 피드백 부족, 로딩 상태 미흡 |
| 모바일 UX | 6.0 | 바텀시트/하단탭 있으나 터치 타겟 작음, 중간 화면 대응 부재 |
| 접근성 | 3.0 | ARIA 속성 부재, 키보드 내비게이션 미흡, 색상 대비 미달 |
| 온보딩/학습성 | 2.0 | 첫 사용자 가이드 전무, 빈 상태 안내 미흡 |
| 스타일 유지보수성 | 2.5 | 100% 인라인 스타일, `<style>` 태그 반복 삽입 |

### 1.2 핵심 문제 TOP 7

#### 문제 1: 인라인 스타일 100% — 유지보수 불가
```jsx
// 동일 스타일이 5개 이상 파일에 복붙
const ls = { display:"block", fontSize:12, fontWeight:500, color:C.tt, marginBottom:6 };
const is = { width:"100%", padding:"9px 12px", borderRadius:8, border:`1px solid ${C.bd}` };
```
- hover, focus, active 등 상태 스타일 표현 불가 → `<style>` 태그로 우회
- 매 렌더링마다 `<style>` 재삽입 → 성능 저하, 깜빡임 가능
- Tailwind v4 설치되어 있으나 전혀 활용 안 됨

#### 문제 2: 학생 상세 페이지 인지 과부하
```
메인탭(4개) x 서브탭(10개) = 최대 3단계 깊이
수업 → 수업이력/캘린더/교재
학습관리 → 숙제/오답/파일
분석 → 성적/SWOT
공유/리포트 → 공유설정/PDF
```
- 사용자가 원하는 정보까지 평균 3번 클릭 필요
- 탭 전환 시 현재 컨텍스트 유실
- 1,238줄 단일 컴포넌트 — 모든 탭이 상태 공유로 전체 리렌더

#### 문제 3: 시각적 계층(Visual Hierarchy) 부족
- 모든 카드/블록/섹션이 동일한 `border: 1px solid #E7E5E4` + `borderRadius: 14`
- 중요도에 따른 시각적 차별화 없음 (미수금 경고가 일반 정보와 같은 무게감)
- 타이포그래피가 13~15px 범위에서 미세한 차이만 존재

#### 문제 4: 로딩/빈/에러 상태 경험 미흡
- **로딩**: "불러오는 중..." 텍스트만 표시 (스켈레톤 UI 없음)
- **빈 상태**: "학생을 추가해보세요" 한 줄 텍스트 (일러스트/CTA 없음)
- **에러**: "데이터를 불러오지 못했습니다" + 다시 시도 버튼만 존재
- **부분 에러**: 5개 API 중 1개 실패 시 전체 에러 처리

#### 문제 5: 아이콘 시스템 부재
- 모든 아이콘이 인라인 SVG로 하드코딩 (약 40개 이상)
- 크기/색상 변경 시 매번 SVG 코드 수정 필요
- Sidebar, AppShell, Students 등에서 동일 아이콘 중복 정의

#### 문제 6: 모바일 경험 단절
- 768px 기준 2단계만 (모바일/데스크톱) → 태블릿 경험 부재
- 터치 타겟: 일부 버튼이 22px x 22px (최소 44px 권장)
- 대시보드 2컬럼 그리드가 모바일에서 1컬럼으로 전환되나 블록 순서 최적화 없음

#### 문제 7: 온보딩/첫 사용자 경험 부재
- 첫 로그인 후 빈 대시보드 표시 → 사용자 이탈 위험
- 복잡한 기능 (반복 수업, SWOT, 공유 설정)에 대한 안내 없음
- 기능 발견성(Discoverability) 낮음 — Ctrl+Z 되돌리기, 드래그 기능 등

### 1.3 잘 된 점 (유지해야 할 것)
- 따뜻한 중성 톤 컬러 팔레트 — #FAFAF9 배경, Pretendard 폰트와 조화
- 학생별 8색 컬러 시스템 — 시간표/학생 목록에서 직관적 구분
- 모바일 하단 네비게이션 — 4개 핵심 메뉴, safe-area 대응
- 커스터마이즈 가능한 대시보드 — 드래그 블록 재배치, 숨기기/복원
- 시간표 인터랙션 — 드래그 생성/이동, 컨텍스트 메뉴, Undo
- 학부모 공유 뷰 — 토큰 기반, 세밀한 권한 설정

## 2. 디자인 시스템 리디자인

### 2.1 컬러 시스템 개선

#### 개선안: 확장된 컬러 토큰 체계

```
[텍스트 — 대비율 개선]
--color-text-primary:    #1A1A1A  (유지, 15.4:1)
--color-text-secondary:  #57534E  (변경: 4.6:1 → AA 충족)
--color-text-tertiary:   #78716C → #6B7280 (변경: 4.8:1 → AA 충족)
--color-text-placeholder:#A3A3A3  (NEW)

[액센트 — 확장]
--color-accent:       #2563EB    (유지)
--color-accent-hover: #1D4ED8    (NEW: 호버 상태)

[그림자 — 깊이 체계 추가]
--shadow-sm:  0 1px 2px rgba(0,0,0,.05)
--shadow-md:  0 4px 12px rgba(0,0,0,.08)
--shadow-lg:  0 12px 32px rgba(0,0,0,.12)
--shadow-xl:  0 20px 60px rgba(0,0,0,.15)
```

### 2.2 타이포그래피 스케일 정립

```
Display:     28px / 800 / -0.5px tracking  — 페이지 타이틀
Heading 1:   22px / 700 / -0.3px tracking  — 섹션 타이틀
Heading 2:   18px / 700 / normal           — 카드/모달 타이틀
Heading 3:   15px / 600 / normal           — 블록 타이틀
Body:        14px / 400 / normal           — 기본 본문
Body Strong: 14px / 600 / normal           — 강조 본문
Caption:     12px / 500 / normal           — 보조 텍스트, 라벨
Micro:       11px / 400 / normal           — 뱃지, 타임스탬프
Tiny:        10px / 600 / 0.5px tracking   — 카테고리 라벨
```

### 2.3 간격(Spacing) 시스템

```
--space-1:  4px    — 아이콘-텍스트 간격
--space-2:  8px    — 인라인 요소 간격
--space-3:  12px   — 컴팩트 패딩
--space-4:  16px   — 기본 패딩
--space-5:  20px   — 카드 내부 패딩
--space-6:  24px   — 섹션 간격
--space-7:  28px   — 페이지 패딩
--space-8:  32px   — 대형 섹션 간격
```

### 2.4 아이콘 시스템 도입

```
lucide-react 도입 이유:
1. 트리쉐이킹 지원 — 사용한 아이콘만 번들에 포함
2. 일관된 24px 그리드, strokeWidth 조절 가능
3. 300+ 아이콘 — 현재 필요한 모든 아이콘 커버

적용 예시:
  <Calendar size={20} strokeWidth={1.8} />    ← 현재: 인라인 SVG 6줄
  <Users size={20} strokeWidth={1.8} />        ← 현재: 인라인 SVG 4줄
```

## 3. 페이지별 UI/UX 개선안

### 3.1 로그인 페이지

| # | 현재 | 개선안 | 우선순위 |
|---|------|--------|:---:|
| 1 | 입력 필드에 `<label>` 미연결 | `<label htmlFor>` + 시각적 라벨 표시 | P1 |
| 2 | 에러 메시지가 폼 하단에 고정 | 해당 필드 바로 아래에 인라인 에러 표시 | P2 |
| 3 | 로딩 시 "처리 중..." 텍스트만 | 버튼 내 스피너 + 텍스트, 입력 비활성화 | P2 |
| 4 | 비밀번호 표시/숨기기 없음 | 눈 아이콘 토글 추가 | P3 |
| 5 | 비밀번호 강도 표시 없음 (회원가입) | 실시간 강도 인디케이터 추가 | P3 |

### 3.2 대시보드

**A. 시각적 계층 재설계**
```
현재: 모든 블록이 동일한 카드 스타일
      ┌─────────┐ ┌─────────┐
      │ Block A │ │ Block B │  ← 전부 같은 외관
      └─────────┘ └─────────┘

개선: 중요도별 차별화
      ┌═══════════════════════┐
      ║  다음 수업 준비 (Hero) ║  ← 그래디언트 배경, 큰 타이포
      ╚═══════════════════════╝
      ┌──────────┐ ┌──────────┐
      │ 미기록 ⚠ │ │ 주의학생 │  ← 경고 색상 강조
      └──────────┘ └──────────┘
```

**B. 빈 대시보드 → 온보딩 경험**
```
현재: 빈 블록들 나열

개선 (첫 사용자):
  ┌──────────────────────────────────────┐
  │  과외 매니저에 오신 것을 환영합니다!   │
  │                                      │
  │  3단계로 시작해보세요:                │
  │  (1) 첫 학생 등록하기      [시작] →   │
  │  (2) 수업 일정 추가하기    [나중에]   │
  │  (3) 수업 기록 남기기      [나중에]   │
  └──────────────────────────────────────┘
```

**C. 스켈레톤 로딩**
```

### 3.3 학생 관리 페이지

**A. 카드 정보 밀도 최적화**

개선 카드:
┌─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─┐

│─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─│
│              [수정] [...]     │  ← 더보기 메뉴로 통합

**B. 뷰 모드 전환**
```

**C. 학생 추가 폼 UX 개선**
```
현재: 9개 필드 한 번에 나열

개선: 2섹션 분리
  ── 기본 정보 (필수) ──
    이름*, 과목*, 학년
  ── 추가 정보 (선택, 접이식) ──
    학교, 연락처, 학부모, 수업료, 단가, 생년월일
```

### 3.4 학생 상세 페이지 (가장 큰 개선 필요)

#### 개선안: 플랫 네비게이션 + 고정 헤더

```
┌──────────────────────────────────────────┐
│  ← 학생 목록                             │
│                                          │
│  [아바타] 김민준                          │
│           수학 · 고2 · 한영고             │
│           다음 수업: 내일 14:00           │
│           [공유링크] [PDF] [...]          │
│──────────────────────────────────────────│
│                                          │
│  수업  숙제  성적  오답  교재  자료  전략  │ ← 1단 수평 탭 (스크롤)
│  ────                                    │
│                                          │
│  [선택된 탭의 콘텐츠]                     │
└──────────────────────────────────────────┘
```

**핵심 변경:**
1. 중첩 탭 제거 → 1단 수평 탭 (스크롤 가능)
2. 학생 프로필 헤더 항상 고정 → 탭 전환 시에도 컨텍스트 유지
3. 각 탭을 독립 컴포넌트로 분리 → lazy loading
4. 주요 액션 버튼 헤더에 배치 → 공유/PDF 항상 접근 가능

## 5. 인터랙션 패턴 표준화

### 5.1 피드백 시스템

| 액션 | 현재 | 개선 |
|------|------|------|
| 데이터 저장 | Toast만 | Toast + 버튼 체크 애니메이션 |
| 삭제 | ConfirmDialog → Toast | + Undo 옵션 (5초) |
| 드래그 | 투명도 변경만 | 그림자 + 확대 + 드롭 영역 하이라이트 |
| 로딩 | "불러오는 중..." | 스켈레톤 UI |
| 에러 | 텍스트 + 재시도 | 에러 유형별 안내 + 해결 방법 |
| 빈 상태 | 텍스트 1줄 | 일러스트 + 설명 + CTA |

### 5.2 모달 → 사이드 패널 (데스크톱)

```
데스크톱에서 수업 상세 보기:
┌──────────────┬──────────────────────┐
│              │                      │
│   기존 페이지 │   사이드 패널         │
│   콘텐츠     │   (수업 상세 보기)    │
│              │                      │
│              │   수업 내용...        │
│              │   피드백...           │
│              │   숙제...            │
│              │                      │
│              │   [저장] [삭제]       │
└──────────────┴──────────────────────┘

모바일에서는 기존 바텀시트 모달 유지
```

### 5.3 폼 인터랙션 표준화

```
입력 필드 상태:
  Default  → 회색 보더 (#E7E5E4)
  Focus    → 파란 보더 (#2563EB) + 파란 그림자
  Error    → 빨간 보더 (#DC2626) + 에러 메시지
  Disabled → 회색 배경 (#F5F5F4)

버튼 상태:
  Default  → 기본 스타일
  Hover    → 약간 어두운 배경 + 미세 그림자
  Active   → 더 어두운 배경
  Loading  → 스피너 + "처리 중..." + 비활성화
  Disabled → 투명도 60% + not-allowed
```

## 6. 모바일 UX 강화

### 6.1 반응형 브레이크포인트 확장

```
현재: 768px 기준 2단계

개선: 4단계
  sm  (<=640px):   모바일
  md  (641~768):   모바일 확장
  lg  (769~1024):  태블릿 — 사이드바 접힘, 2컬럼
  xl  (>1024):     데스크톱 — 사이드바 펼침
```

### 6.2 터치 타겟 표준화

```
현재 문제:
  - 대시보드 편집 버튼: ~28px 높이 (부족)
  - 이동 버튼: 22x22px (매우 부족)
  - 바텀 내비 라벨: 10px (너무 작음)

기준:
  - 최소 터치 타겟: 44x44px
  - 버튼 최소 높이: 36px (모바일 44px)
  - 터치 타겟 간 최소 간격: 8px
```

### 6.3 모바일 전용 개선

```
1. 스와이프 제스처:
   - 학생 카드 좌스와이프 → 빠른 액션 (수정/보관/삭제)
   - 수업 카드 좌스와이프 → 빠른 기록

2. Pull-to-Refresh:
   - 대시보드, 학생 목록, 시간표에서 당겨서 새로고침

3. 바텀시트 모달 확장:
   - 모든 모달을 모바일에서 바텀시트로 표시
   - 드래그 핸들 + 스냅 포인트

4. 모바일 FAB (Floating Action Button):
   - 우하단 고정 버튼: 빠른 수업 기록 / 학생 추가
```
